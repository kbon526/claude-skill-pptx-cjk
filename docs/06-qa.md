# 06 — 双轨 QA：结构验证 + 视觉核查

**永远不要假设第一次构建就对**。本文档的两条 QA 路径是经过多个项目反复验证的最低安全线。

## QA 哲学

1. **假设有问题，去找它** — 不是"确认一切都好"，是 bug hunt 心态
2. **结构 + 视觉双轨**，缺一不可
3. **不要相信肉眼第一遍** — 看过自己代码的人，会"看见想看见的"，不是"实际显示的"
4. **修一个 bug 后必须重检** — 一个修复经常引入新问题

## 轨 1：结构 QA（程序化）

用 [`scripts/qa_deep.py`](../scripts/qa_deep.py) 做全自动结构检查。

### 检查项

| 检查项 | 容差 | 说明 |
|--------|------|------|
| 形状边界越出 slide | ±0.05 英寸 | 任何 shape 的左/右/上/下超出 slide 边界即报错 |
| 文本溢出 textbox | 130% 高度 | 估算文字占用高度 > textbox 高度 130% 即警告 |
| 标题 / 页脚冲突 | 相交 | title 占位符与 footer 占位符重叠即报错 |
| 标题字数过长 | >18 字 | 内容页标题超长警告 |
| 字号未定义 | run.font.size = None | 提示该 run 字号靠继承（不稳定）|

### 用法

```bash
$ python3 scripts/qa_deep.py output.pptx
```

输出示例：
```
=== Slide 04 项目背景 ===
  ✗ Shape "Rectangle 5" 右边界 13.45 in 超出 slide (13.33 in)
  ⚠ Shape "TextBox 12" 文本预估高度 1.85 in > textbox 1.30 in (溢出 142%)
  ✗ Title placeholder 与 footer placeholder 在 (Inches 6.8, 7.2) 区域重叠

=== Slide 09 KPI 框架 ===
  ⚠ Title text "我们将通过多渠道整合实现品牌升级与销量突破的双重目标" (28字) 超长

Total: 2 errors, 2 warnings, 19 slides clean
```

### CJK 文本溢出公式

```python
def estimate_text_height_inches(text, font_pt, textbox_width_in,
                                 line_spacing=1.2):
    cjk = sum(1 for c in text if '一' <= c <= '鿿')
    latin = len(text) - cjk
    # 单字符宽度（英寸）
    cjk_w = font_pt / 72
    latin_w = font_pt * 0.55 / 72
    total_w = cjk * cjk_w + latin * latin_w
    line_count = max(1, total_w / textbox_width_in)
    line_h = font_pt * line_spacing / 72
    return line_count * line_h
```

公式来自 BambuLab 项目实战，对微软雅黑准确度 ~90%。

## 轨 2：视觉 QA（PIL 渲染备用 + LibreOffice 主路径）

### 主路径：LibreOffice 转 PDF + pdftoppm 转图片

```bash
soffice --headless --convert-to pdf output.pptx
pdftoppm -jpeg -r 150 output.pdf slide
# 生成 slide-01.jpg, slide-02.jpg, ...
```

然后**人眼**或**让 Claude 多模态**检查每张图。Claude 多模态检查时给一个明确清单：

```
请逐张检查以下幻灯片图片，假设每张都有问题。
查找：
- 元素重叠（文字穿过形状、线条压字）
- 文本溢出 / 边界裁切
- 标题下装饰线在标题换行后位置错乱
- 注脚或页码与正文太近（< 0.3 英寸）
- 元素过近（卡片几乎贴边）
- 间距不均（一处大块留白，另一处拥挤）
- 边距不足（< 0.5 英寸）
- 列对齐不一致
- 低对比度文字（浅灰文字 + 米色背景这种）
- 残留模板占位符文字（"标题 1"、"Lorem ipsum"）

逐张列出问题或可疑点，即使是小问题也要列。
```

**每次对话用一个新的 Agent / subagent 看图**。自己已经盯了半天代码，会"自动美化"不存在的瑕疵或忽略明显的问题。

### 备用路径：PIL 渲染几何边界

如果 LibreOffice 不可用（沙箱受限、headless 超时），用 [`scripts/render_layout.py`](../scripts/render_layout.py) 直接画 shape 边界图：

```bash
python3 scripts/render_layout.py output.pptx qa_layout/
```

输出：
- `qa_layout/layout-01.png` 到 `layout-NN.png`
- 80 px / 英寸分辨率
- **绿框** = placeholder（关键布局锚点）
- **灰框** = 自由形状
- 红字显示 textbox 内文字（前 90 字符）
- 底部蓝色字标注 slide 编号 + layout 名

PIL 渲染的优点：
- 不依赖任何系统级渲染服务
- 在沙箱、CI、远程服务器都能跑
- 能直接看出占位符位置、形状重叠、边界越界

缺点：
- 不渲染真实字体、颜色、阴影（只是几何）
- 看不出最终视觉效果

## 检查文本是否有残留占位符

模板用多了，容易留下"请输入标题"、"Click to edit"、"Lorem ipsum"等残留：

```bash
python3 -m markitdown output.pptx | grep -iE "xxxx|lorem|ipsum|click.*edit|请输入|此处填写"
```

或更狠的：

```bash
python3 -m markitdown output.pptx | grep -E "^\s*$|TODO|占位"
```

## QA 验证循环

```
1. 生成 → 2. 转图 → 3. 检查 → 4. 列出问题清单 → 5. 修
                                              ↑
                                              └─ 重新生成对应 slides 再检查
```

**绝对不要"修了一个就声明完成"**。修后必须重检该页和相邻页。

## 交付前的最终 checklist

- [ ] 跑过 `qa_deep.py`，errors 数 = 0，warnings ≤ 3 且都已确认
- [ ] 转 PDF 全 deck 翻一遍，每页停留 ≥ 5 秒
- [ ] 用 Claude 多模态过一遍，至少一轮无新增问题
- [ ] 检查残留占位符（grep 命令）
- [ ] 验证字体（在另一台机器上打开，看是否回退）
- [ ] 验证文件可在 PowerPoint Windows 版打开
- [ ] 文件名规范（无中文括号、无 emoji）
- [ ] 文件大小合理（< 30MB，否则邮件附件会被拦）

## QA 反例

❌ "我看了一遍没问题" — 没跑 qa_deep.py，没用 subagent 看图
❌ "改了第 5 页就好了" — 没回头检查第 6 页（很可能也有同样问题）
❌ "客户那边没问题就行" — 客户不是 QA，他们没义务找 bug
